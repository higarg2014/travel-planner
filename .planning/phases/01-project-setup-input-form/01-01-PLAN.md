  <action>
Build the trip input form with all required fields using Material-UI components.

**Create src/types/trip.ts:**
Define TypeScript interface for trip input:
```typescript
export interface TripInput {
  destination: string;
  checkIn: Date | null;
  checkOut: Date | null;
  travelers: number;
  budget: number;
  preferences: {
    activities: string;
    hotelType: string;
    foodPreferences: string;
  };
}
```

**Create src/components/TripInputForm.tsx:**
Build form component with Material-UI:
1. Import Material-UI components: TextField, Button, Box, Typography, Container, Grid2, CircularProgress
2. Import DatePicker from @mui/x-date-pickers and LocalizationProvider from @mui/x-date-pickers/LocalizationProvider
3. Import AdapterDayjs from @mui/x-date-pickers/AdapterDayjs
4. Use useState to manage form state (TripInput interface)
5. Use useState for loading state (boolean)
6. Create form with these fields (all in Material-UI Container with max-width md, padding, centered):
   - **Destination:** TextField with label "Destination", placeholder "e.g., Paris, France", fullWidth
   - **Check-in Date:** DatePicker with label "Check-in Date", minDate today
   - **Check-out Date:** DatePicker with label "Check-out Date", minDate checkIn + 1 day, disabled if no check-in selected
   - **Number of Travelers:** TextField with type="number", label "Number of Travelers", InputProps min 1, default 1
   - **Budget:** TextField with type="number", label "Total Budget ($)", InputProps min 0, InputProps startAdornment "$"
   - **Preferences section:** Typography variant h6 "Preferences (Optional)", then:
     - TextField label "Activities" placeholder "e.g., museums, hiking, beaches" multiline rows 2
     - TextField label "Hotel Type" placeholder "e.g., boutique, luxury, budget"
     - TextField label "Food Preferences" placeholder "e.g., vegetarian, local cuisine"
   - **Submit button:** Button variant="contained" size="large" fullWidth, shows CircularProgress when loading, text "Generate Itinerary"
7. Form layout: Use Grid2 for responsive layout (spacing 3), cards with Paper component for visual hierarchy
8. Form submission handler: preventDefault, set loading true, console.log form data, setTimeout 2 seconds then set loading false (placeholder for Phase 2 API call)
9. Basic validation: Disable submit if destination empty, checkIn null, checkOut null, travelers < 1, or budget <= 0

**Update src/App.tsx:**
1. Import ThemeProvider and createTheme from @mui/material
2. Import CssBaseline from @mui/material
3. Import LocalizationProvider and AdapterDayjs
4. Wrap app in ThemeProvider with default theme
5. Add CssBaseline for Material-UI baseline styles
6. Wrap in LocalizationProvider with AdapterDayjs
7. Render TripInputForm component
8. Add Typography variant h3 title "AI Trip Planner" centered above form
9. Clean layout: centered, good spacing, Material-UI's responsive Container

Keep styling minimal (default Material-UI, no custom theme for speed). Focus on functionality and clean Material-UI components.
